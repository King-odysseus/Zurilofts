Key Concepts in Plain English

What is a "Component"?

A React component is a reusable piece of UI. Think of it like a blueprint. PropertyCard.jsx takes property data and renders the same card layout every time, just with different content. You write it once and use it everywhere.

What is an "API Endpoint"?

An endpoint is a URL that the frontend calls to get or save data. GET /api/properties returns all properties as JSON. The frontend says "give me properties" and the backend responds with an array of property data.

What is "Middleware"?

Middleware runs between the request arriving and the response being sent. The authenticate middleware checks if the user is logged in. The validate middleware checks if the data sent is in the correct format.

Request Flow (What Happens When You Click a Button)

1

User clicks "Book Now" on a property page

The React component calls apiClient.post('/bookings', bookingData)

2

Axios sends HTTP POST to /api/bookings

Vite proxies this to localhost:3000/api/bookings. The auth interceptor attaches the user's JWT token.

3

Express matches the route

server/src/routes/booking.routes.ts matches POST / to bookingCtrl.create

4

Middleware runs

authenticate verifies the JWT. validate(bookingCreateSchema) checks all fields are valid.

5

Controller handles the request

server/src/controllers/booking.controller.ts extracts data from req.body, calls the service.

6

Service does the actual work

server/src/services/booking.service.ts checks availability, calculates pricing, creates the database record.

7

Response goes back

Controller sends { success: true, data: booking } back to the browser. Axios resolves the promise. React updates the UI.

Never use native <select>. Always use the Dropdown component from src/components/Dropdown.jsx. This keeps the fly-out menu consistent with the app's design.

Adding Features

Step-by-step guides for common changes

Example: Add a New Admin Page

Let's add a "House Rules" admin page that lets the admin configure rules.

Step 1: Create the React Page

src/pages/AdminHouseRules.jsx (new file)
import { useState } from 'react';

function AdminHouseRules() {
  const [rules, setRules] = useState('');
  const [saved, setSaved] = useState(false);

  async function handleSave(e) {
    e.preventDefault();
    // Save to API
    setSaved(true);
  }

  return (
    <div className="w-full">
      <h1 className="text-2xl font-bold text-[#262262] mb-6">House Rules</h1>
      <div className="bg-white rounded-2xl shadow-sm border border-[#D9D9D9] p-6">
        <form onSubmit={handleSave}>
          <textarea
            value={rules}
            onChange={(e) => setRules(e.target.value)}
            className="w-full px-4 py-3 rounded-xl border border-[#D9D9D9] focus:outline-none focus:border-[#C49A6C] h-40"
            placeholder="Enter house rules..."
          />
          <button type="submit" className="mt-4 bg-[#C49A6C] text-white font-semibold px-6 py-2.5 rounded-full hover:bg-[#b8895c]">
            Save Rules
          </button>
          {saved && <span className="ml-3 text-green-600 text-sm">Saved!</span>}
        </form>
      </div>
    </div>
  );
}
export default AdminHouseRules;

Step 2: Add the Route

src/App.jsx (add inside the admin Route group)

Troubleshooting

Common problems and how to fix them

ProblemFix
"Module not found" errorRun npm install in that folder
Database errors / missing columnsRun cd server && npx prisma db push
Prisma client out of dateRun cd server && npx prisma generate
Port 3000 already in useKill the process: taskkill /f /im node.exe (Windows) or change PORT in .env
Port 5173 already in useVite will auto-pick the next available port
White page in browserOpen browser dev tools (F12) → Console tab for errors
API calls failing (401 Unauthorized)Your login token expired. Sign out and sign in again.
Images not showingCheck server/uploads/ folder exists and has files. Check the image path starts with /uploads/.
Google sign-in not workingCheck GOOGLE_CLIENT_ID + GOOGLE_CLIENT_SECRET in server/.env are correct. The callback URL must match Google Cloud Console settings.
Changes not appearing in browserHard refresh: Ctrl+Shift+R. If that fails, restart both dev servers.
tailwind.config.js changes not applyingRestart the Vite dev server (Terminal 2). Tailwind only reads config at startup.
Backend TypeScript errors on buildRun cd server && npx tsc --noEmit to see all type errors. Fix them before building.
npm install fails with permissionsTry npm install --legacy-peer-deps or delete node_modules folder and try again.
Can't push to GitHubMake sure you're on a branch: git checkout -b your-branch-name. Then push: git push -u origin your-branch-name.
.env file not loadingServer reads server/.env, not the root .env. Make sure it's in the server folder.
